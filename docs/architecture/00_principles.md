# 設計原則

Rv Meta Code は、データベース定義から OpenAPI / SDK / メタデータを生成するデスクトップアプリケーションである。設計の主目的は、長期間の実装・保守を前提にした、AI でも扱いやすい構造を維持することにある。

## 1. 基本思想

- 責務を明確に分離する
- モジュール間の依存を単方向にする
- AI による実装が既存構造を崩さないようにする
- テストしやすい設計にする
- 将来的な機能追加を容易にする

## 2. レイヤードアーキテクチャ

本プロジェクトでは MVC ではなく、次のレイヤー構造を採用する。

```text
Presentation
    ↓
Application
    ↓
Domain
    ↓
Infrastructure
```

依存関係は、常に上位レイヤーから下位レイヤーへの一方向とする。

## 3. 目指す品質

- 変更範囲が局所化されること
- 画面・ロジック・データアクセスの責務が分離されていること
- 依存関係が明示され、回帰テストしやすいこと
- 生成物やメタデータ管理が一貫した構成で保守されること
