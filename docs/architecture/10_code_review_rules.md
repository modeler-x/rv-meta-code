# コードレビュー規約

## 1. レビューの観点

コードレビューでは、以下を重視する。

- 変更の意図が明確であること
- 責務が適切に分離されていること
- 命名が説明的であること
- 依存関係が一方向であること
- テストが追加・更新されていること

## 2. 指摘の優先順位

1. 仕様との齟齬
2. アーキテクチャ違反
3. 保守性の低下
4. テスト不足
5. 命名・スタイルの問題

## 3. AI 実装時の注意点

- 既存構造を壊さない
- 300 行超えや責務の混在を避ける
- 既存ルールと整合した実装を行う
- 変更範囲が大きい場合は、段階的に分割してレビューしやすくする
